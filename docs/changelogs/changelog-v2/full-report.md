# VTaxon 改動整理（changelog-v1 → changelog-v2）

以功能面向分類，依使用者體感影響排序。

---

## 2. 物種搜尋 UX 改進

### 品種攔截面板 & 中文名稱回報
- **統一物種搜尋 UX**：新增品種攔截面板，搜尋到有品種的物種時自動引導使用者選擇品種
- 新增**中文名稱回報功能**：使用者發現物種中文名有誤時可直接回報
- 修正 NameReportForm 下拉選單從品種結果中正確提取父物種
- 修正 name-reports API 三個 bug

### 搜尋無結果引導
- 搜尋無結果時提供「**找不到物種**」回報表單，引導使用者提交回報而非放棄
- 搜尋引導 UX 三輪迭代：checkbox 文案改善、CJK 提示、學名即時驗證、驗證分流、disabled 樣式
- 大幅減少無效的「物種不存在」回報

### 品種回報表單
- 品種回報表單重設計 — **強制搜尋流程**，確保使用者先搜尋再回報

### 中文搜尋修正
- 中文名判斷改用 **CJK 字元檢測**，避免拉丁學名被誤判為中文名
- 中文名查詢改為**外部 API 優先**，修正快取過時導致物種中文名錯誤
- 中文搜尋加入本地 species_cache 查詢，解決「學名能找到但中文名搜不到」的問題
- species_cache 快取更新擴展至所有核心分類欄位，修正路徑錯誤
- 修正 TaiCOL taxon_group 對不存在的輸入回傳全部資料庫的問題
- 修正屬名中文名稱錯誤取得動物界的問題

## 3. 品種 Grid 排列（分類樹 UI）

### 多排 Grid 佈局
- 品種節點改為**多排 Grid 排列**，解決同層品種橫向無限成長的問題
- breed grid 依**子樹寬度動態分配**每排品種數
- breed grid 後自動重跑 compactTree，消除祖先節點殘留空白

### 連線與排序
- 連線改為**主幹 + 分支模式**，取代每排獨立 bezier 曲線
- 主幹移到最左欄左側，避免與子樹交錯
- 主幹與水平線避開品種節點標籤
- 排序改為按 **VTuber 數量升序**，數量少的品種排在上面

### 其他修正
- breed grid 只對品種節點生效，修正分類樹其他線條錯亂
- 品種面板「不指定品種」按鈕上移
- AI 分類 prompt 全面更新
- 家鴨（Anas platyrhynchos f. domesticus）中文名稱 override

## 4. 虛構物種

### 分類結構修正
- 虛構物種溯源範圍支援 **4 段 category_path** + 標籤重命名
- 虛構物種樹計數排除 type node，只計算**葉節點**
- 補上「民間精怪」3 段 type node 記錄，使分類層新增按鈕正常顯示
- 同步虛構物種 seed 與 DB，修正資料損壞

### 新增虛構物種（4 個）
- **睡魔**（Sandman）— 西方神話 / 日耳曼傳說（修正 category_path 中間分類改為「民間精怪」避免與葉節點重名）
- **索普德特**（Sopdet）— 埃及神話（天狼星神格化身）
- **附魂體**子分類重整：搬移 Doll 並新增 **Cursed Doll**
- **Teddy Bear** 從「機械生命」搬移到「附魂體」

### 表單改善
- 虛構物種回報表單新增**字數限制**
