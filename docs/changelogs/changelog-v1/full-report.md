# VTaxon 上線後改動整理（3/6 23:30 ~ 3/10 16:30）

以功能面向分類，依使用者體感影響排序。

---

## 1. 物種搜尋與中文名稱顯示

### 俗名（別名）顯示
- 搜尋結果新增**俗名（別名）**顯示：從 TaiCOL 和 Wikidata 取得物種的其他中文名稱，獨立一行完整展示（不截斷）
- 俗名資料清洗：過濾掉屬名、科名、非中文字元、與主名重複的項目；修正全形括號；Unicode NFC 正規化

### 中文名修正
- 新增**物種中文名覆蓋表**（SPECIES_ZH_OVERRIDES），解決 Wikidata 回傳錯誤中文名的問題
  - 已覆蓋：兔猻、前突三角龍、毒茄蔘、西部灰松鼠、狼（Canis lupus 原本被 Wikidata 回傳「家犬」）、皇帝企鵝、家豬、紅爪雨林蠍
- 修正**種級中文名誤帶「屬」後綴**的問題（Wikidata 回傳屬級名稱時自動去除），統一在 SpeciesCache model 層處理，搜尋結果和樹狀圖都適用
- 修正中文名快取邏輯：寫入 species_cache 前驗證是否為 CJK 字元，避免快取到拉丁文
- 修正亞種的 path_zh.species 未套用父物種覆蓋表的問題（例如 Canis lupus 的亞種在樹狀圖仍顯示「家犬」）

### 搜尋能力擴充
- 支援**高階分類群搜尋**（如水母亞門 Medusozoa、缽水母綱 Scyphozoa 等），不再限於種級
- 新增 **GBIF `/species/match` 後備查詢**：當 suggest 無結果時自動 fallback
- 新增 **TaiCOL 後備查詢**：當 GBIF 完全無結果時（如 Medusozoa 類群）改查 TaiCOL
- 修正**同義詞（synonym）自動解析**：搜尋到 GBIF 同義詞時自動導向 accepted name
- 修正同義詞大量回傳導致搜尋超時的問題
- TaiCOL 中文搜尋改為先查 common_name 再查 taxon_group，提高命中率
- 搜尋框提示文字更新，建議使用拉丁文搜尋以提高準確度

### 新增分類階層支援
- 支援 **FORM**（型）分類階層
- 支援 **SUBPHYLUM**（亞門）和 **SUBCLASS**（亞綱）的標籤和中文路徑
- **亞種路徑標準化**：SUBSPECIES / VARIETY / FORM 統一為 8 段路徑格式

### 物種名稱覆蓋（學名）
- 新增**學名覆蓋機制**：當 GBIF accepted name 不是主流用名時，可指定替代學名（如兔猻）

## 2. 分類樹瀏覽（圖鑑頁面）

### 樹展開行為
- **自動展開單子鏈**：路徑上只有一個子節點時自動展開，不會在每一層都要手動點
- 子節點 ≤ 5 時**遞迴自動展開**，減少點擊次數
- 修正樹展開邏輯：父節點 > 5 子但子節點 ≤ 5 時，子節點也會遞迴展開
- 虛構物種樹預設不自動全展開（避免畫面過於複雜）
- 調整預設展開層級

### 樹佈局與視覺
- 修正**格狀排列連線在攝影機下移時消失**的問題
- 修正樹節點佈局：subtree-extent-based compaction 取代舊的 recenterParents，解決節點重疊和空隙問題
- 學名去除作者引用（如 "Linnaeus, 1758"），只顯示純學名
- **合併重複物種節點** + 新增「未指定品種」節點（只在該物種確實有品種子節點時才顯示）
- 支援 `__breed__unspecified` 路徑，HUD 定位和邊線高亮都正確運作

### 鏡頭定位
- 修正登入用戶的**初始鏡頭定位**：等 auth 解析完成後再載入樹，確保鏡頭能正確對焦到使用者的位置
- 延遲鏡頭定位直到 expandedSet 穩定，避免定位到尚未展開的節點
- 修正 toast 通知觸發樹資料刷新時鏡頭跳動的問題
- 修正 Supabase token 更新和 user object 變更導致樹收合 / 重置的問題

### 排序與顯示
- 預設排序改為**建立時間倒序（新的在前）**
- 側欄使用者列表也改為新建在前
- 使用群組排序時，樹節點上顯示**國旗和組織徽章**

### 行動版 UI
- 重新設計**行動版浮動工具列**：改為水平頂部列 + 抽屜式 FocusHUD
- 排序和樹切換改為 inline 操作，減少點擊步驟

## 3. 虛構物種分類重整

### 類型層（Type Layer）
- 新增第三層分類「**類型（Type）**」：分類結構從 origin → sub_origin → species 擴充為 origin → sub_origin → **type** → species
- 管理後台支援 Type 層設定 + 路徑衝突檢測
- Type 節點可選取為 trait（新增按鈕）
- 分類修正：狼人從不死族移至魔物、偽人（Doppelgänger）歸入魔物

### 新增虛構物種（共 19 個）

#### 批次處理 #1（migration 20260309，17 個新物種 + 新增凱爾特神話 sub_origin）
- **奇幻文學 → 通用**（4 筆）：夢魔（Incubus）、曼德拉草（Mandrake）、樹人（Ent）、半精靈（Half-elven）
- **西方神話 → 希臘神話**（3 筆）：卡戎（Charon）、半神（Demigod）、樹精（Dryad）
- **西方神話 → 歐洲民間傳說**（3 筆）：所羅門魔神（Goetic Demon）、雙足飛龍（Wyvern）、死神（Grim Reaper）
- **西方神話 → 凱爾特神話**（1 筆，新 sub_origin）：費伊（Fey）
- **現代虛構 → 克蘇魯神話**（1 筆）：外神（Outer God）
- **東方神話 → 中國神話**（1 筆）：盤古（Pangu）
- **人造生命 → 機械生命**（2 筆）：泰迪熊（Teddy Bear）、機械鳥（Mechanical Bird）
- **非物質生命 → 能量態生命**（2 筆）：植物精靈（Plant Spirit）、時間精靈（Chrono Spirit）

#### 分類重整時同步新增（2 個）
- 四足飛龍 Dragon (Four-legged)（歐洲民間傳說 → 西方龍）
- 墮天使 Fallen Angel（奇幻文學 → 通用 → 天界）

#### 個別新增
- 陸吾（Lu Wu）（東方神話 → 中國神話 → 神靈）— 3/7
- 山藥怪物（Yam Monster）（東方神話 → 台灣民間傳說）— 3/10

## 4. 通知系統

- **全新通知系統**：管理員審核品種申請 / 虛構物種申請後，系統自動通知申請者
- **多階段通知流程**：已收到 → 處理中（已排入待辦）→ 已完成 / 不處理
- 已讀 / 未讀視覺區分優化（未讀更明顯）
- 管理員將申請轉為「處理中」時也發送通知
- 修正批量通知插入的 UUID 型別錯誤

## 5. 管理後台

### 申請管理
- 新增**匯出功能**：品種申請和虛構物種申請可匯出為 JSON 檔案下載
- 新增**批量狀態轉移 API**：一次處理多筆申請
- 管理員可查看各類申請的**待處理數量**

### 用詞調整
- 「駁回」→「未採納」→ 最終統一為「**不處理**」，語氣更溫和
- 「已受理」→「**已排入待辦**」，避免使用者誤解為已在處理
- 管理員備註和通知訊息保留換行格式

### 其他
- 修正 admin 頁面因 Supabase token 更新導致自動刷新的問題
- 匯出 JSON 移除多餘的 reference_data 欄位
