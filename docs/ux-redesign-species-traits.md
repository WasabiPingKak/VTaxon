# 真實物種特徵設定 — UX 重新設計討論紀錄

> 本文件記錄 UX 重設計的完整討論脈絡與決策，供後續實作參考。完成後刪除。

---

## 一、現有頁面盤點

| 頁面 | 路由 | 功能 |
|------|------|------|
| 角色設定 | `/profile?tab=species` | 查看/新增/刪除自己的物種特徵，內嵌 SpeciesSearch |
| 搜尋頁 | `/search` | 獨立搜尋頁，搜到可直接加特徵（無品種選擇） |
| 品種目錄 | `/breeds` | 手風琴品種瀏覽、全域品種搜尋、回報缺漏品種 |
| 公開檔案 | `/vtuber/:id` | 別人看到的角色頁面（唯讀） |

### 核心元件

- **SpeciesSearch** (`frontend/src/components/SpeciesSearch.jsx`)：最複雜的元件，處理物種+品種搜尋、結果分組、亞種懶載入、品種快捷按鈕、搜尋引導
- **SettingsRealSpecies** (`frontend/src/components/settings/SettingsRealSpecies.jsx`)：`/profile` 內的物種特徵管理 UI
- **BreedsPage** (`frontend/src/pages/BreedsPage.jsx`)：品種目錄頁，包含 BreedRequestInline（回報缺漏）、GlobalBreedSearch、BreedAccordion

## 二、發現的問題

### 問題 1：品種目錄頁角色不明確

`/breeds` 同時想當：
- 品種瀏覽器（手風琴目錄）
- 品種搜尋引擎（全域搜尋）
- 品種缺漏回報入口（BreedRequestInline）
- 甚至能直接「加入特徵」（跳過 `/profile` 流程）

**結論**：品種目錄作為獨立頁面價值低，應併入主搜尋流程。

### 問題 2：加入特徵有太多入口

`/profile`、`/search`、`/breeds` 三個地方都能加特徵，體驗不一致（有的能選品種，有的不能）。

### 問題 3：品種選擇沒有整合進物種選擇流程

理想流程：選物種 → 如果有品種就選品種。目前品種是獨立的發現路徑。

### 問題 4：使用者分不清品種與物種

大部分使用者不理解分類學層級，會直接輸入品種名（如「柴犬」、「Shiba Inu」）當作物種名搜尋。搜不到就以為系統沒有，直接填回報單，產生大量無效回報。

### 問題 5：中文搜尋斷裂（技術 bug）

**現象**：用學名搜尋能正確帶出中文名，但用同一個中文名搜尋反而找不到物種。

**根因**：
- 學名搜尋路徑：GBIF suggest → 命中 → 從 Wikidata/TaiCOL 抓中文名 → 寫入 `species_cache.common_name_zh`
- 中文搜尋路徑：只查 TaiCOL API（台灣物種名錄）→ 很多物種不在 TaiCOL 裡 → 查無
- **本地 `species_cache` 已經有中文名，但中文搜尋完全不查本地快取**

**修復方案**：中文搜尋加入 Phase 1 本地 `species_cache` 查詢（`WHERE common_name_zh LIKE '%query%'`），在 TaiCOL 之前。

### 問題 6：缺少中文名稱錯誤/缺漏回報管道

**現象**：物種存在但中文名缺漏或錯誤，使用者無處回報。

目前的回報機制只有：
- 品種回報 (`breed_requests`) — 品種不存在
- 虛構物種回報 (`fictional_species_requests`) — 虛構物種不存在
- 帳號檢舉 (`user_reports`) — 使用者違規

## 三、設計決策

### 回報缺漏的嚴格 gate 是刻意的

> 「太多人只用中文搜尋找不到就回報，其實正確使用學名就能搜到。透過 UI/UX 設計強迫他們真的試過學名搜尋，系統裡真的沒有才回報。」

回報流程要求「至少搜尋過一次非 CJK 查詢」的 gate 保留，但入口應該在使用者遇到問題的 context 內（搜尋結果為空時），而非獨立頁面。

### 簡化後的 User Story

| # | User Story | 處理方式 |
|---|-----------|---------|
| A | 我要設定角色的物種（和品種） | 統一搜尋：物種+品種同時搜，選物種後如果有品種就展示品種選擇 |
| B | 系統沒有我要的品種，想回報 | 回報入口移到搜尋結果 context 內，保留強制學名搜尋 gate |
| C | 物種的中文名缺漏或錯誤，想回報 | **新增**回報管道，入口在搜尋結果的物種旁 |
| D | 純瀏覽品種目錄 | **砍掉**，併入 A 的搜尋流程 |

### 統一搜尋的流程設計

```
使用者輸入任何東西（柴犬 / Shiba Inu / Canis lupus / 貓）
  │
  ├─ 搜尋結果分兩區呈現：
  │   ┌─────────────────────────────────┐
  │   │ 🏷️ 品種                         │
  │   │  柴犬 Shiba Inu                 │
  │   │  → 屬於 家犬 Canis lupus fam.   │  ← 明確顯示父物種
  │   │  [新增]                          │
  │   ├─────────────────────────────────┤
  │   │ 🔬 物種                         │
  │   │  灰狼 Canis lupus               │
  │   │  家犬 Canis lupus familiaris     │
  │   │  [新增] → 如果有品種，展開品種選擇 │
  │   └─────────────────────────────────┘
  │
  └─ 搜尋無結果時：
      ┌─────────────────────────────────┐
      │ 找不到「xxx」                     │
      │                                  │
      │ 💡 試試用拉丁學名搜尋            │
      │ [________________] [重新搜尋]     │
      │                                  │
      │ 確定系統沒有？→ [回報缺漏]        │
      └─────────────────────────────────┘
```

### `/breeds` 頁面 → 砍掉

品種瀏覽、品種搜尋、回報缺漏全部收進 SpeciesSearch，不再需要獨立頁面。
